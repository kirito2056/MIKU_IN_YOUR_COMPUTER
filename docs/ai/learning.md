# 학습 파이프라인 (Learning Pipeline)

## 1. 학습 전략 (성격 안정 우선)

-   **점진적 학습은 하지 않음.** 대화 로그를 모델에 계속 학습시키면 성격 코어가 희석될 수 있으므로, 성격은 **코어 데이터만**으로 파인튜닝하고 고정한다.
-   **개인화·맥락 반영**은 파인튜닝이 아니라 **기억 검색(메모리)**으로 한다. 수정이 필요하면 **기억을 가져오는 방식**(키워드/벡터 검색, L2·L3 활용)을 개선하는 쪽으로 한다.

### A. 로깅 (기억용)
1.  **Logging**: 대화 로그 실시간 저장 → L1(단기) → L2(중기, Vector DB) → L3(장기, Vault)로 이관.
2.  **용도**: 이 로그는 **파인튜닝용이 아님**. 추론 시 **기억 검색**으로 프롬프트에 넣어 맥락·개인화를 준다.

### B. 파인튜닝 (성격 고정용)
-   **시점**: 성격 매트릭스(코어 데이터)가 바뀌었을 때, 또는 주기적으로 성격만 갱신하고 싶을 때 **수동** 실행.
-   **데이터**: 성격 코어 데이터셋만 사용. 대화 로그는 넣지 않음.
-   **Backup**: 학습 완료된 LoRA는 Google Drive 등에 버전별 백업 (선택).

## 2. 자연 성장 시스템 (Natural Growth)
-   **Hidden Stats**:
    -   친밀도, 지식량, 시각 능력, 유머 감각.
-   **Mechanism**:
    -   스탯창 없음. 대화를 통해 자연스럽게 해금되는 기능들.
    -   예: 시각 능력 Lv.5 -> 작은 글씨 읽기 가능.

## 3. Tech Teacher & Knowledge
-   **IT 선생님**: 모르는 용어 질문 시 칠판 꺼내서 설명.
-   **Konami Code**: 히든 커맨드 입력 시 이스터 에그 발동.

## 4. Debugging (투명한 뇌)
-   **Debug Mode**: 화면 구석에 미쿠의 사고 과정(Chain of Thought)을 텍스트로 출력.
