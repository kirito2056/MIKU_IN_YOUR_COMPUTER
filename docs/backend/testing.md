# 테스트 전략 (Testing Strategy)

## 1. 테스트 접근 방식

파인튜닝 완료 후 구체적인 성능 수치를 결정하되, 기본적인 테스트 프레임워크는 미리 구축합니다.

## 2. 엣지 케이스 자동화 테스트

주요 엣지 케이스를 자동으로 테스트하여 시스템 안정성을 보장합니다.

### 2.1. 테스트 시나리오

#### A. LLM 추론 실패 시나리오
-   **시나리오**: LLM 응답 타임아웃 (5초 초과)
-   **예상 동작**: Emergency Model로 자동 전환
-   **검증**: 응답이 정상적으로 생성되는지 확인

#### B. GPU 메모리 부족
-   **시나리오**: VRAM 90% 이상 사용 시 추가 요청
-   **예상 동작**: 작업 대기 또는 다른 GPU로 분산
-   **검증**: 시스템이 크래시하지 않고 적절히 처리하는지 확인

#### C. WebSocket 연결 끊김
-   **시나리오**: 네트워크 연결 강제 종료
-   **예상 동작**: 자동 재연결 (최대 10회, 5초 간격)
-   **검증**: 재연결 후 대화 컨텍스트 복원 확인

#### D. 플러그인 크래시
-   **시나리오**: 플러그인 프로세스 강제 종료
-   **예상 동작**: 플러그인 비활성화, 메인 시스템 정상 동작
-   **검증**: 메인 시스템이 영향받지 않는지 확인

#### E. DB 연결 실패
-   **시나리오**: 데이터베이스 서버 다운
-   **예상 동작**: 재연결 시도, 실패 시 WARNING 로그
-   **검증**: 시스템이 계속 동작하되 적절한 로그 기록

### 2.2. 자동화 실행
-   **스케줄**: 매일 밤 (수면 모드 진입 전) 또는 수동 실행.
-   **도구**: Python `pytest` 또는 `unittest`.
-   **리포트**: 테스트 결과를 로그 파일로 저장.

## 3. 성능 테스트

### 3.1. 목표
-   **체감되지 않을 정도의 성능**: 사용자가 지연을 느끼지 않을 수준.
-   **구체적 수치**: 파인튜닝 완료 후 실제 측정값을 기준으로 결정.

### 3.2. 모니터링 항목 (임계값 기반)

#### 프레임 드롭
-   **모니터링**: 3D 렌더링 프레임레이트 추적.
-   **임계값**: 특정 프레임 이하로 떨어지면 경고 (구체적 수치는 파인튜닝 후 결정).
-   **대응**: 프레임 드롭 감지 시 자동으로 렌더링 품질 조정 또는 일시 중지.

#### CPU 온도
-   **모니터링**: CPU 온도 실시간 추적.
-   **임계값**: 특정 온도 이상 시 경고 (구체적 수치는 파인튜닝 후 결정).
-   **대응**: "CPU 온도 너무 높아요" 알림 + 로그 기록.

#### GPU 온도
-   **모니터링**: 각 GPU 온도 실시간 추적.
-   **임계값**: 특정 온도 이상 시 경고 (구체적 수치는 파인튜닝 후 결정).
-   **대응**: "오빠, 나 머리가 너무 뜨거워..." 알림 + 로그 기록.

### 3.3. 응답 시간 테스트
-   **음성 응답**: STT -> LLM -> TTS 전체 파이프라인 시간 측정.
-   **목표**: 1초 이내 (요구사항 문서 참조).
-   **측정**: 파인튜닝 완료 후 실제 환경에서 벤치마크.

## 4. 통합 테스트

### 4.1. 주요 사용 시나리오
-   **대화**: 음성 입력 -> 응답 생성 -> 음성 출력 전체 플로우.
-   **그림 생성**: "그림 그려줘" 요청 -> Stable Diffusion 실행 -> 이미지 표시.
-   **플러그인 사용**: 플러그인 설치 -> 기능 실행 -> 결과 확인.

### 4.2. 사용자 시나리오 테스트
-   **일반 대화**: 다양한 주제로 대화하여 응답 품질 확인.
-   **장시간 사용**: 연속 대화로 메모리 누수 확인.
-   **동시 작업**: 여러 기능 동시 사용 시 리소스 관리 확인.

## 5. 테스트 도구

### 5.1. Python 테스트 프레임워크
-   **단위 테스트**: `pytest` 또는 `unittest`.
-   **통합 테스트**: `pytest` with fixtures.
-   **모킹**: `unittest.mock` for 외부 의존성.

### 5.2. 성능 프로파일링
-   **Python**: `cProfile`, `line_profiler`.
-   **GPU**: `nvidia-smi` 로그 분석.
-   **메모리**: `memory_profiler`.

## 6. 테스트 데이터

### 6.1. 대화 데이터셋
-   **일반 대화**: 다양한 주제의 대화 샘플.
-   **엣지 케이스**: 긴 문장, 특수 문자, 이모지 등.
-   **성격 테스트**: 미쿠의 성격이 잘 반영되는지 확인하는 대화.

### 6.2. 시뮬레이션 데이터
-   **하드웨어 스트레스**: 높은 부하 상황 시뮬레이션.
-   **네트워크 이슈**: 연결 끊김, 지연 시뮬레이션.
