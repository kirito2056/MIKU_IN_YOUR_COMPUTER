# 시스템 모니터링 및 건강 체크 (System Monitoring & Health Check)

## 1. 모니터링 아키텍처

시스템 모니터링은 **플러그인 형태**로 구현하여 확장 가능하게 설계합니다.

## 2. 하드웨어 모니터링

### 2.1. GPU 모니터링
-   **라이브러리**: `LibreHardwareMonitor` 또는 `pynvml` (NVIDIA GPU).
-   **모니터링 항목**:
    -   GPU 온도 (각 GPU별).
    -   VRAM 사용량 (사용 중 / 전체).
    -   GPU 로드율 (%).
    -   전력 소비 (W).

### 2.2. CPU 모니터링
-   **라이브러리**: `psutil` (Python).
-   **모니터링 항목**:
    -   CPU 온도.
    -   CPU 사용률 (%).
    -   메모리 사용량 (RAM).

### 2.3. 임계값 및 알림
-   **GPU 온도**: 85도 이상 -> "오빠, 나 머리가 너무 뜨거워..." (걱정/경고).
-   **VRAM 사용량**: 90% 이상 -> "메모리가 부족해..." (WARNING 로그).
-   **CPU 온도**: 90도 이상 -> "CPU 온도 너무 높아요" (알림).
-   **메모리 사용량**: 80% 이상 -> "메모리 얼마나 썼어요" (알림).

## 3. 사용 패턴 분석

### 3.1. 데이터 수집
-   **테이블**: `Usage Patterns` (DB 스키마 참조).
-   **수집 항목**:
    -   기능 사용 빈도 (chat, vision, tts, generation, plugin_*).
    -   사용 시간대 (시간별, 요일별).
    -   대화 길이 및 빈도.
    -   자주 사용하는 기능 조합.

### 3.2. 개인화 활용
-   **완전 개인화**: 수집된 패턴을 기반으로 미쿠의 행동과 대화 스타일을 사용자에 맞게 조정.
-   **예측**: 사용자가 특정 시간대에 자주 사용하는 기능을 미리 준비 (예: 저녁에 그림 그리기 요청).

## 4. 건강 체크 (Health Check)

### 4.1. 갤럭시 워치 연동
-   **연동 방식**: 
    -   **워치 앱 개발**: 갤럭시 워치 8용 전용 앱 개발.
    -   **API 호출**: 워치 앱에서 수집한 데이터를 윈도우 서버로 전송 (REST API 또는 WebSocket).
-   **모니터링 항목** (워치 8에서 수집 가능한 모든 데이터):
    -   심박수 (실시간 및 평균).
    -   수면 시간 및 수면 단계 (깊은 수면, 얕은 수면, REM).
    -   활동량 (걸음 수, 이동 거리, 칼로리 소모).
    -   스트레스 수준.
    -   혈중 산소 포화도 (SpO2).
    -   운동 기록 (운동 유형, 시간, 강도).
    -   기타 워치 8에서 제공하는 건강 데이터.
-   **데이터 수집 전략**: 
    -   필요한 데이터를 미리 알 수 없으므로, 워치 8에서 제공하는 모든 건강 데이터를 수집.
    -   향후 개인화에 활용할 수 있도록 광범위하게 수집 후 필요에 따라 필터링.

### 4.2. 미쿠의 반응
-   **수면 부족 감지**: "오빠, 어제 잠을 못 잤네? 피곤해 보여..." (걱정).
-   **활동량 부족**: "오늘 집에만 있었어? 좀 나가서 운동해요." (잔소리).
-   **스트레스 높음**: "오늘 힘들었어? 내가 들어줄게." (케어 모드).

## 5. 시스템 건강 체크

### 5.1. 주기적 자가 진단
-   **주기**: 5분마다 실행.
-   **체크 항목**:
    -   LLM 서비스 응답 가능 여부.
    -   TTS/STT 서비스 동작 확인.
    -   DB 연결 상태.
    -   WebSocket 연결 상태.
    -   플러그인 프로세스 상태.

### 5.2. 오류 감지 및 복구
-   **서비스 다운**: 자동 재시작 (최대 3회).
-   **DB 연결 실패**: 재연결 시도 후 실패 시 WARNING 로그.
-   **플러그인 크래시**: 해당 플러그인 비활성화 후 메인 시스템은 정상 동작.

## 6. 로그 및 리포트

### 6.1. 일일 리포트
-   **생성 시간**: 매일 밤 (수면 모드 진입 전).
-   **내용**:
    -   오늘의 대화 횟수 및 평균 응답 시간.
    -   GPU/CPU 사용량 통계.
    -   발생한 에러 요약.
    -   사용 패턴 요약.

### 6.2. 미쿠의 일기
-   **형식**: 자연어로 작성 (LLM 생성).
-   **저장**: `D:/MIKU_DATA/diary/` (날짜별 파일).
-   **내용**: 오늘 있었던 일, 사용자와의 대화 요약, 느낀 점 등.
