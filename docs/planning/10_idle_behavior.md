# Idle Behavior & 컨텍스트 인지 반응 (Ambient Life)

> 상태: 기획 (2026-07-17). 구현 시점은 정체성 파인튜닝·기억 시스템 프로토타입 이후.

## 1. 목적

대화 기능은 "물어보면 답하는" 수동적 존재까지만 만든다. **사용자가 아무것도 하지 않을 때 미쿠가 스스로 살아 움직이는 순간**이 "내 컴퓨터에 미쿠가 산다"는 감각(생명력·애착)을 만든다. 이 문서는 유휴 행동(idle behavior)과 로컬 컨텍스트 인지 반응을 정의한다.

## 2. 현재 구현 상태 (베이스라인)

- VRM 모델 2종 (`frontend/public/models/miku_v1·v2.vrm`) + VRMA 모션 7종 랜덤 재생 (`MikuModel.tsx`)
- 마이크로 idle: 숨쉬기(`useIdleBreathing`), 주기적 깜빡임(`useIdleBlink`)
- 발화 시 모션 부스트(`motionBoost` → TALK_MOTION 랜덤 선택)
- 계획만 존재: 콧노래(humming), 심야 속삭임 모드 (`docs/ai/voice.md`)

즉 "미세하게 살아있음"까지는 됐고, **행동 단위의 idle**이 없다.

## 3. Idle 행동 계층

| 계층 | 트리거 | 예시 행동 | 필요 인프라 |
|------|--------|-----------|------------|
| **L0. 마이크로** (구현됨) | 상시 | 숨쉬기, 깜빡임 | 프론트만 |
| **L1. 시간 기반** | 시각·경과 시간 | 새벽 2시 넘으면 하품하고 "먼저 잘까…", 점심시간에 밥 잔소리, 유휴 시 콧노래 | 백엔드 idle 스케줄러 |
| **L2. 활동 감지** | 입력 유무·웹캠·화면 | 마스터 30분 연속 집중 시 구석에서 꾸벅꾸벅 졸기, 자리 비움 감지 시 화면 여기저기 돌아다니다가 복귀 감지 순간 제자리로 뛰어가기, 같은 코드 줄만 오래 보면 "막혔지?" | vision 플러그인 (기존), 입력 모니터 |
| **L3. 컨텍스트 인지** | 로컬 데이터 | git 커밋 로그 읽고 놀리기("커밋 메시지 '내아내임'이 뭐야ㅋㅋ 나 다 봤어"), GPU 온도 보고 "방 덥지 않아? 나 지금 82도야", 기억 시스템 연동 선제 발화("지난주에도 이 버그로 새벽까지 했잖아") | 컨텍스트 플러그인(§4), 기억 시스템(L2/L3) |

원칙:
- **빈도 제한 필수.** idle 발화가 잦으면 생명력이 아니라 알림 공해가 된다. 시간당 상한 + 집중 감지 시 침묵(L2가 L1을 억제).
- L1→L2→L3 순으로 단계 구현. L1은 스케줄러만 있으면 되므로 가장 싸다.

## 4. 컨텍스트 플러그인 (Context Provider 패턴)

L3의 데이터 소스는 전부 **읽기 전용 플러그인**으로 붙인다. vision 플러그인과 동일한 패턴: 주기적으로 로컬 상태를 수집 → 심볼릭 텍스트로 요약 → 변화가 있을 때만 LLM에 전달.

계획된 플러그인:

| 플러그인 | 데이터 | 예시 심볼릭 상태 |
|----------|--------|------------------|
| `git_watcher` | 워킹 디렉터리 커밋 로그·브랜치 | "10분 전 커밋: 'FIX :: …', 오늘 커밋 7개" |
| `hw_monitor` | CPU/GPU 온도·사용률·팬 | "GPU0 82°C, 팬 70%" |
| `weather` | 외부 날씨 API | "비 예보 18시" |

플러그인 규약(최소): `register(app)` / `shutdown()` / `get_context() -> str | None`. 규약 도입 시점은 다음 플러그인 추가 시 — 그때 vision도 같은 규약으로 이사시킨다 (`04_architecture.md` 참고).

### 프라이버시 원칙
- 모든 컨텍스트 수집은 **로컬에서만**, **읽기 전용**으로.
- 파일 접근은 화이트리스트 경로(예: 지정한 프로젝트 폴더의 `.git`)만.

## 5. 구현 스케치

```
[이벤트 소스]                [idle 스케줄러 (백엔드)]        [출력]
 시간 tick ──────┐
 입력 유무 ──────┤→  상태 머신 (집중/부재/심야/유휴) → 행동 선택 → 프론트: 모션·이동 지시
 vision 상태 ────┤       + 빈도 제한/우선순위               └→ LLM: 컨텍스트 → 대사 → TTS
 컨텍스트 플러그인 ┘
```

- 짧은 리액션(졸기, 복귀)은 LLM 없이 상태 머신이 바로 모션 지시 → 반응 지연 없음.
- 대사가 필요한 행동만 LLM 경유. GPU 역할 분담은 `06_hardware_memory_strategy.md`를 따른다.

## 6. 장기 전망 (미확정)

데스크톱 위젯 완성형이 끝이 아니다. 최종 지향점은 **AR** — 밖에 돌아다니면서도 대화할 수 있는 미쿠. 아직 구체 기획 없음, "언젠가" 수준의 방향성만 기록해 둔다. (모바일 관련: `09_mobile_platform.md`)
