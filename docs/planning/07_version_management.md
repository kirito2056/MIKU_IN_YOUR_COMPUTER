# 버전 관리 전략 (Version Management)

## 1. 버전 체계 (Versioning Scheme)

현재 버전 관리 전략은 미정입니다. 향후 필요 시 다음 체계를 고려할 수 있습니다:

### 제안: Semantic Versioning (SemVer)
-   **형식**: `MAJOR.MINOR.PATCH` (예: `1.0.0`)
-   **MAJOR**: 호환되지 않는 API 변경, 대규모 아키텍처 변경.
-   **MINOR**: 새로운 기능 추가 (하위 호환성 유지).
-   **PATCH**: 버그 수정, 작은 개선.

## 2. 업데이트 전략 (Update Strategy)

### 2.1. 자동 업데이트
-   **Library**: `electron-updater` 사용.
-   **Flow**:
    1.  GitHub Releases 주기적 체크.
    2.  새 버전 발견 시 백그라운드 다운로드.
    3.  업데이트 완료 후 재시작 시: "오빠, 나 좀 달라진 거 없어?" (대사).

### 2.2. 데이터 호환성
-   **DB 스키마 변경 최소화**: 버전 업그레이드 시 기존 데이터와 호환되도록 설계.
-   **마이그레이션**: 필수 스키마 변경 시 자동 마이그레이션 스크립트 실행.
-   **롤백 지원**: 업데이트 실패 시 이전 버전으로 자동 롤백 (선택 사항).

## 3. 모델 버전 관리

### 3.1. AI 모델 관리
-   **모델 준비**: 
    -   기본 모델 수동 다운로드
    -   파인튜닝 수행 (별도 프로세스)
    -   파인튜닝 완료된 모델 경로 지정
-   **로컬 저장**: `D:/MIKU_DATA/models/`
-   **버전 관리**: Google Drive에 별도 저장
    -   **트리거**: AI 버전 업그레이드 또는 모델 변경 시
    -   **포맷**: LoRA 어댑터 + 성격 메타데이터 + 메타데이터 JSON 압축
    -   **파일명**: `miku_v{version}_{date}.zip`

### 3.2. 모델 호환성
-   **로딩 실패 시**: 이전 버전 모델로 자동 폴백.
-   **성능 비교**: 새 모델과 이전 모델의 응답 품질 비교 로그 저장.

## 4. 설정 및 메타데이터

### 4.1. 성격 설정 (Personality Metadata)
-   **저장 방식**: 버전별 JSON 파일로 관리.
    -   파일명: `personality_v{version}.json` (예: `personality_v1.0.json`)
    -   위치: `D:/MIKU_DATA/personality/`
    -   내용: 성격 매트릭스 전체 설정 (docs/planning/02_personality_matrix.md 기반)
-   **버전 관리**:
    -   성격 매트릭스 변경 시 새 버전 JSON 파일 생성.
    -   버전별 변경 이력 추적 가능.
-   **백업 전략**:
    -   파인튜닝된 모델과 함께 압축하여 Google Drive에 저장.
    -   압축 파일 구조: `miku_v{version}_{date}.zip`
        -   `model/`: LoRA 어댑터 파일
        -   `personality/`: 성격 메타데이터 JSON 파일
        -   `metadata.json`: 모델 정보, 학습 날짜, 데이터셋 정보 등

### 4.2. 사용자 설정
-   **호환성**: 버전 업그레이드 시 사용자 설정 자동 마이그레이션.
-   **백업**: 설정 파일은 D드라이브에 별도 백업.
